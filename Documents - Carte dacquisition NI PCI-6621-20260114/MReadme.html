M Series Additional Documentation Readme

This readme provides an overview of the files included on the M Series Additional Documentation for NI-DAQ 7.3 CD. It also provides supplemental documentation to help you program an M Series device with NI-DAQmx.

CD Contents

The M Series Additional Documentation for NI-DAQ 7.3 CD contains the following:

  • M Series Documentation—Keep the following documents in the same directory so that all links work:

M Series Help—This help file explains the features and components common to all M Series devices. It includes pinouts for supported M Series devices and specifications.

Specifications—These documents provide detailed specifications for NI 622x and NI 625x devices. Specifications for NI 628x devices are available at ni.com/manuals.

Supported Properties/Attributes—This file contains the all the supported M Series properties/attributes broken down by device.

  • .bin3 Example Files—These files contain updates for the NI Example Finder needed for M Series devices to be displayed. To view updated devices in the NI Example Finder, replace the existing .bin3 file with the appropriate .bin3 file on the CD as follows:

For LabVIEW 7.0, replace the existing .bin3 file, which is installed by default at National Instruments\Shared\Example Finder\1.0\Products\LabVIEW\7.0\exbins\english\platform\daqmxw.bin3, with the .bin3 file of the same name on the CD.

For LabVIEW 7.1, replace the existing daqmxw.bin3 and daq72mxw.bin3 files, which are installed by default at National Instruments\Shared\Example Finder\1.0\Products\LabVIEW\7.1\exbins\english\platform, with the .bin3 files of the same name on the CD.

For LabWindows/CVI 7.0, replace the existing cvidaqmxw.bin3 file, which is installed by default at National Instruments\Shared\Example Finder\1.0\Products\CVI\7.0\exbins\english\platform, with the .bin3 file of the same name on the CD.

M Series Valid Sources for the Analog Trigger

APFI 0 and APFI 1

When configuring an analog trigger, connect your analog signal to either the APFI 0 or APFI 1 terminal and specify APFI 0 or APFI 1 as your trigger source.

Analog Input Channel

In addition to APFI 0 and APFI 1, analog input tasks can trigger off of one of the analog input channels being sampled. Because M Series devices use a scanning architecture, many restrictions are placed on how you can use an analog trigger when the source is one of the channels you are sampling. When you use an Analog Start Trigger, the trigger channel must be the first channel in the channel list. When you use an Analog Reference or Pause Trigger, and the analog channel is the source of the trigger, there can be only one channel in the channel list. If you have more than one channel for Pause or Reference Triggers, you must use APFI 0 or APFI 1.

M Series Physical Channels

Dev1 in physical channel names is the default device name for M Series devices. You can change these names in MAX.

Analog Input

A 16-channel M Series device has physical channels ranging from Dev1/ai0 to Dev1/ai15. You can configure only channels 0 through 7 in differential mode. When you configure a channel in differential mode, the channel is the positive input and channel plus eight is the negative input. For instance, if you configure channel 1 in differential mode, the positive input is channel 1, and channel 9 is the negative input.

A 32-channel M Series device has physical channels ranging from Dev1/ai0 to Dev1/ai31. You can configure channels 0-7 and 16-23 as the positive channel of a differential pair. If N is this channel, channel N + 8 is the negative input of the pair.

Use only the physical channel name of the positive channel (not both) when creating a differential channel.

Analog Output

An M Series device that supports two analog outputs has analog output physical channels named Dev1/ao0 and Dev1/ao1.

An M Series device that supports four analog outputs has analog output physical channels named Dev1/ao0, Dev1/ao1, Dev1/ao2, and Dev1/ao3.

Digital Input and Output

All M Series devices have eight or 32 lines of digital input and output named Dev1/port0/line0 through Dev1/port0/line7 or Dev1/port0/line0 through Dev1/port0/line31. These lines belong to a single port, and the physical channel Dev1/port0 refers to all eight or 32 lines at once. Port 0 can perform both hardware-timed and static digital operations.

M Series devices have two more ports, port 1 and port 2. Port 1 has eight digital I/O lines, Dev1/port1/line0 through Dev1/port1/line7. Port 2 has eight digital I/O lines, Dev1/port2/line0 through Dev1/port2/line7. Port 1 and port 2 can be used as static digital I/O lines or PFI lines, PFI <0..15>. When any of PFI lines <0..15> is used as a digital I/O signal, it uses the physical channel name shown in the following table.

Counter Input and Output

All M Series devices have two counter/timers referred to by the physical channel names Dev1/ctr0 and Dev1/ctr1. Unlike the other I/O types, these physical channel names do not refer to terminals on the I/O connector but instead to circuits within the device. There are four primary terminals associated with each counter. These are the terminals used as the SOURCE, GATE, AUX, and OUT functions. NI-DAQmx has default values for these terminals. For counter input tasks, if you know whether your signal provides the SOURCE, AUX, or GATE function and wire your signal to the default, you do not have to set the Input Terminal attribute/property.

M Series AO External Reference Sources

For some M Series devices, you can use APFI 0 or APFI 1 for AO external reference sources. Refer to the specifications for your device for additional information.

M Series Default Input/Output Terminal Configurations

The default terminal configuration is differential for channels 0 through 7 and 16 through 23. The default is referenced single-ended for channels 8 through 15 and 24 through 31.

The default output terminal configuration is referenced single-ended.

M Series Calibration

Your device uses software calibration to adjust the software scaling of signals read from and produced by your device. Using calibration PWMs with a reference voltage, your device measures and calculates scaling constants for analog input and analog output. The scaling constants are stored in nonvolatile memory (EEPROM) on your device. To achieve specification accuracy, you should self-calibrate your device just before a measurement session but after your computer and the device have been powered on and warmed up for at least 15 minutes. You should allow this same warm-up time before performing any calibration of your system. Frequent calibration produces the most stable and repeatable measurement performance. The device is not harmed in any way if you recalibrate it often.

Calibrating your device takes some time. Do not be alarmed if the DAQmx Self-Calibrate or DAQmx Adjust M Series Calibration function/VI takes several seconds to execute.

For best results, stop any ongoing tasks and disconnect any unnecessary external connections before running calibration.

M Series Calibration Signal Connections

Self-Calibration

When you self-calibrate your M Series device, no signal connections are necessary. However, values generated on the analog output channels change during the calibration process. If you have external circuitry connected to the analog output channels and you do not want changes on these channels, you should disconnect the circuitry before beginning the self-calibration.

External Calibration

When externally calibrating your M Series device, make the signal connections described for the type of M Series device you are calibrating. Set the reference voltage between +6.000 V and +8.5 V. Typically, you use a calibrator or other stable voltage source for the reference voltage. Do not use a power supply as its signals are not very stable.

Follow these steps for M Series devices. Refer to Connecting Analog Input Signals in the NI-DAQmx Help for more on connecting signals:

  1. Disconnect any external connections or circuitry to your M Series device.
  2. Connect the positive output of your reference voltage source to physical channel ai0.
  3. Connect the negative output of your reference voltage source to physical channel ai8.
  4. If your reference voltage source and your computer are floating with respect to each other, connect the negative output of your reference voltage source to the AI GND terminal as well as to physical channel ai8.
  5. Run the external calibrate VI.

Events

The AI Hold Complete Event is supported on M Series devices. This event is a signal emitted by an E Series or M Series analog input circuit when the analog signal at the physical channel being measured has been latched or held. The AI Hold Complete Event signals an external multiplexer to switch to the next channel. This signal was previously known as SCANCLK, which is the legacy name of the external terminal where this signal can be emitted.

Refer to Events in the NI-DAQmx Help for other supported events.

Position Measurements with an Encoder

With an M Series device, you can measure position with an encoder, such as a two-pulse encoder or quadrature encoder. M Series devices support three types of decoding for quadrature encoders: X1, X2, and X4. M Series devices also support the Z index, which provides a precise reference point and is available on some encoders. In addition to encoders, you can use an LVDT or an RVDT to measure position, with appropriate signal conditioning circuitry, such as an SCXI-1540 module. For additional information on position measurements, refer to Measuring Position in the NI-DAQmx Help.

M Series Device Terminals

M Series devices are not currently included in the NI-DAQmx Help, nor in the Measurement & Automation Explorer Help for NI-DAQmx. Refer to the M Series Help, included on this CD, for pinouts.

Duplicate Count Prevention

M Series devices support duplicate count prevention, a way to prevent incorrect duplicate counts caused by a timebase being slower than the signal you want to measure. Refer to Duplicate Count Prevention in the NI-DAQmx Help for additional information.

API Changes for M Series

A new calibration function/VI has been added to NI-DAQ 7.3 for M Series devices. M Series devices also support some new properties/attributes. Refer to Suppported Properties/Attributes for M Series Devices for a complete list.

Calibration Function/VI for M Series

To adjust the external calibration constants for an M Series device, use the DAQmx Adjust M-Series Calibration function/VI. To adjust constants, you must connect a known voltage to the device and specify that voltage with reference voltage. Refer to the help for DAQmx Adjust E-Series Calibration in the LabVIEW Help or in the NI-DAQmx C Function Reference Help for the specific parameters you set.

In LabVIEW, you can find DAQmx Adjust M-Series Calibration VI by opening the LabVIEW directory (for instance, Program Files\National Instruments\LabVIEW 7.0) and going to vi.lib\calibration\calibrationInfo.llb. Select DAQmx Adjust M-Series Calibration.

Known Issue

If you are using an SCXI-1349 adapter with an M Series device, you need revision B of the SCXI-1349 for parallel mode to work.
